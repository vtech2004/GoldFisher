# 黄金钓鱼 - 网络通信协议文档

## 1. 协议概述

### 1.1 传输层
- **协议**：TCP
- **编码**：UTF-8
- **端口**：默认 8888

### 1.2 消息分帧
由于 TCP 是流式协议，需自定义消息边界。采用**换行符分隔法**：

```
[JSON消息体]\n
```

- 每条消息以 UTF-8 编码的 JSON 字符串表示
- 每条消息以换行符 `\n` 结尾
- 接收方按 `\n` 拆分流式数据，逐条解析 JSON

### 1.3 消息格式
所有消息均为 JSON 格式：

```json
{
  "type": "消息类型",
  "data": { ... }
}
```

- `type`：字符串，消息类型标识
- `data`：对象，消息具体内容（不同消息结构不同）

### 1.4 服务器响应通用结构
```json
{
  "type": "响应类型",
  "data": {
    "success": true,
    "message": "操作成功",
    ...其他业务字段
  }
}
```

- `success`：布尔值，表示操作是否成功
- `message`：结果描述文本
- 错误时返回 `ErrorResponse` 类型，包含 `message` 和 `requestType` 字段

## 2. 消息定义

### 2.1 login - 登录

客户端连接服务器后发送的第一个消息，用于玩家身份认证。

#### 请求
```json
{
  "type": "login",
  "data": {
    "playerName": "FisherKing"
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| playerName | string | 是 | 玩家名，2-20字符 |

#### 响应
```json
{
  "type": "login_resp",
  "data": {
    "code": 0,
    "msg": "success",
    "playerId": "p_10001",
    "playerName": "FisherKing",
    "totalScore": 3500,
    "highestLevel": 3,
    "bombs": 0
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0成功，非0失败 |
| msg | string | 结果描述 |
| playerId | string | 服务器分配的玩家唯一ID |
| playerName | string | 玩家名 |
| totalScore | int | 累计总积分 |
| highestLevel | int | 已通过的最高关卡数（0表示尚未通关任何关卡） |
| bombs | int | 当前持有炸弹数（保留字段，默认0） |

---

### 2.2 level_result - 关卡结果上报

玩家完成一关后，客户端上报关卡结果。

#### 请求
```json
{
  "type": "level_result",
  "data": {
    "playerId": "p_10001",
    "level": 1,
    "score": 700,
    "itemScore": 550,
    "timeBonus": 150,
    "timeUsed": 45,
    "passed": true,
    "items": [
      { "name": "小鱼", "count": 3, "value": 50 },
      { "name": "中鱼", "count": 2, "value": 100 },
      { "name": "金币袋", "count": 1, "value": 200 }
    ]
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| playerId | string | 是 | 玩家ID |
| level | int | 是 | 关卡编号（1-5） |
| score | int | 是 | 本关总分（物品分+时间奖励） |
| itemScore | int | 是 | 物品得分合计 |
| timeBonus | int | 是 | 时间奖励分数 |
| timeUsed | int | 是 | 使用时间（秒） |
| passed | bool | 是 | 是否通关 |
| items | array | 否 | 钓到的物品明细列表 |
| items[].name | string | - | 物品名称 |
| items[].count | int | - | 数量 |
| items[].value | int | - | 单个价值 |

#### 响应
```json
{
  "type": "level_result_resp",
  "data": {
    "code": 0,
    "msg": "success",
    "totalScore": 4200,
    "highestLevel": 2,
    "rank": 5
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0成功，非0失败 |
| msg | string | 结果描述 |
| totalScore | int | 更新后的累计总积分 |
| highestLevel | int | 更新后的最高通关关卡 |
| rank | int | 当前全服排名 |

---

### 2.3 player_data - 玩家数据查询

查询指定玩家的详细数据。

#### 请求
```json
{
  "type": "player_data",
  "data": {
    "playerId": "p_10001"
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| playerId | string | 是 | 玩家ID |

#### 响应
```json
{
  "type": "player_data_resp",
  "data": {
    "code": 0,
    "msg": "success",
    "playerId": "p_10001",
    "playerName": "FisherKing",
    "totalScore": 4200,
    "highestLevel": 2,
    "levelRecords": [
      { "level": 1, "bestScore": 700, "passed": true, "playCount": 3 },
      { "level": 2, "bestScore": 1450, "passed": true, "playCount": 2 },
      { "level": 3, "bestScore": 2050, "passed": false, "playCount": 1 }
    ],
    "lastLogin": "2026-06-25 14:30:00"
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0成功，非0失败 |
| msg | string | 结果描述 |
| playerId | string | 玩家ID |
| playerName | string | 玩家名 |
| totalScore | int | 累计总积分 |
| highestLevel | int | 最高通关关卡 |
| levelRecords | array | 各关卡记录 |
| levelRecords[].level | int | 关卡编号 |
| levelRecords[].bestScore | int | 该关最高分 |
| levelRecords[].passed | bool | 是否通关过 |
| levelRecords[].playCount | int | 游玩次数 |
| lastLogin | string | 最后登录时间（yyyy-MM-dd HH:mm:ss） |

---

### 2.4 leaderboard - 排行榜

查询全服积分排行榜。

#### 请求
```json
{
  "type": "leaderboard",
  "data": {
    "playerId": "p_10001",
    "topN": 10
  }
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| playerId | string | 是 | 请求者玩家ID（用于返回自身排名） |
| topN | int | 否 | 返回前N名，默认10，最大100 |

#### 响应
```json
{
  "type": "leaderboard_resp",
  "data": {
    "code": 0,
    "msg": "success",
    "myRank": 5,
    "myScore": 4200,
    "entries": [
      { "rank": 1, "playerId": "p_10003", "playerName": "SeaMaster", "totalScore": 12000 },
      { "rank": 2, "playerId": "p_10005", "playerName": "DeepHook", "totalScore": 9800 },
      { "rank": 3, "playerId": "p_10002", "playerName": "WaveRider", "totalScore": 8500 },
      { "rank": 4, "playerId": "p_10004", "playerName": "HookPro", "totalScore": 6000 },
      { "rank": 5, "playerId": "p_10001", "playerName": "FisherKing", "totalScore": 4200 }
    ]
  }
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 0成功，非0失败 |
| msg | string | 结果描述 |
| myRank | int | 请求者自身排名（0表示未上榜） |
| myScore | int | 请求者自身积分 |
| entries | array | 排行榜列表（按积分降序） |
| entries[].rank | int | 排名 |
| entries[].playerId | string | 玩家ID |
| entries[].playerName | string | 玩家名 |
| entries[].totalScore | int | 累计积分 |

## 3. 错误码定义

| 错误码 | 名称 | 说明 |
|--------|------|------|
| 0 | SUCCESS | 成功 |
| 1001 | INVALID_MESSAGE | 消息格式错误（JSON解析失败） |
| 1002 | UNKNOWN_TYPE | 未知的消息类型 |
| 1003 | MISSING_FIELD | 缺少必填字段 |
| 2001 | LOGIN_FAILED | 登录失败 |
| 2002 | INVALID_NAME | 玩家名不合法（长度/字符） |
| 2003 | NAME_TOO_LONG | 玩家名超长（>20字符） |
| 2004 | NAME_TOO_SHORT | 玩家名过短（<2字符） |
| 3001 | PLAYER_NOT_FOUND | 玩家不存在 |
| 3002 | LEVEL_INVALID | 关卡编号非法 |
| 3003 | LEVEL_NOT_UNLOCKED | 关卡未解锁 |
| 3004 | SCORE_INVALID | 分数非法 |
| 4001 | NOT_LOGGED_IN | 未登录 |
| 4002 | SESSION_EXPIRED | 会话过期 |
| 5001 | INTERNAL_ERROR | 服务器内部错误 |
| 5002 | DB_ERROR | 数据库错误 |
| 5003 | SERVICE_UNAVAILABLE | 服务不可用 |

### 错误响应示例
```json
{
  "type": "login_resp",
  "data": {
    "code": 2004,
    "msg": "玩家名过短，至少2个字符"
  }
}
```

## 4. 通信流程示例

### 4.1 登录流程
```
Client                                    Server
  │                                          │
  │──── TCP Connect ────────────────────────→│
  │                                          │
  │──── login {playerName:"FisherKing"} ────→│
  │                                          │
  │←── login_resp {code:0, playerId:...} ───│
  │                                          │
```

### 4.2 关卡上报流程
```
Client                                    Server
  │                                          │
  │  (玩家完成第1关)                          │
  │                                          │
  │──── level_result {level:1, score:700} ─→│
  │                                          │
  │←── level_result_resp {code:0, rank:5} ──│
  │                                          │
```

### 4.3 查看排行榜流程
```
Client                                    Server
  │                                          │
  │──── leaderboard {topN:10} ──────────────→│
  │                                          │
  │←── leaderboard_resp {entries:[...]} ────│
  │                                          │
```

## 5. 消息类型汇总

| 消息类型 | 方向 | 说明 |
|----------|------|------|
| login | C→S | 登录请求 |
| login_resp | S→C | 登录响应 |
| level_result | C→S | 关卡结果上报 |
| level_result_resp | S→C | 关卡结果响应 |
| player_data | C→S | 玩家数据查询 |
| player_data_resp | S→C | 玩家数据响应 |
| leaderboard | C→S | 排行榜查询 |
| leaderboard_resp | S→C | 排行榜响应 |

> C→S：客户端到服务器；S→C：服务器到客户端

---

## 6. 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 服务器端口 | 8888 | TCP监听端口 |
| 读超时 | 30秒 | 无数据接收超时 |
| 写超时 | 10秒 | 发送数据超时 |
| 心跳间隔 | 15秒 | 客户端心跳周期（保留） |
| 最大消息长度 | 64KB | 单条消息最大字节数 |
