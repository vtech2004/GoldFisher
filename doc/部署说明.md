## 2. 服务器启动步骤

### 2.1 编译服务器

```powershell
# 进入服务器目录
cd e:\Miracle\GoldFisher\FishServer

# 还原依赖
dotnet restore

# 编译
dotnet build -c Release
```

### 2.2 配置服务器

服务器配置文件位于 `FishServer/Config/serverconfig.json`（首次运行时自动生成），默认内容：

```json
{
  "host": "0.0.0.0",
  "port": 8888,
  "maxConnections": 100,
  "dataDir": "data",
  "logLevel": "Info"
}
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| host | 0.0.0.0 | 监听地址（0.0.0.0表示所有网卡） |
| port | 8888 | 监听端口 |
| maxConnections | 100 | 最大连接数 |
| dataDir | data | 数据存储目录（相对路径） |
| logLevel | Info | 日志级别（Debug/Info/Warn/Error） |

### 2.3 启动服务器

```powershell
# 方式一：直接运行
cd e:\Miracle\GoldFisher\FishServer
dotnet run -c Release

# 方式二：运行编译产物
cd e:\Miracle\GoldFisher\FishServer\bin\Release\net8.0
.\FishServer.exe
```

启动成功后控制台输出：
```
[INFO] GoldFisher Server starting...
[INFO] Listening on 0.0.0.0:8888
[INFO] Server started. Press Ctrl+C to stop.
```

### 2.4 验证服务器
使用 telnet 或其他TCP工具连接 `127.0.0.1:8888` 验证服务是否正常。

## 3. 客户端构建步骤

### 3.1 打开项目

1. 打开 Unity Hub
2. 点击"添加项目" → "从磁盘添加"
3. 选择 `e:\Miracle\GoldFisher\FishApp` 目录
4. 确保使用 Unity 2022.3.34f1c1 版本打开

### 3.2 配置服务器连接

客户端配置文件位于 `FishApp/Assets/Config/ServerConfig.json`：

```json
{
  "host": "127.0.0.1",
  "port": 8888
}
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| host | 127.0.0.1 | 服务器IP地址 |
| port | 8888 | 服务器端口 |

> **部署到不同环境时，请修改此配置文件指向正确的服务器地址。**

### 3.3 在Unity编辑器中运行

1. 打开 `Assets/Scenes/Login.unity` 场景
2. 点击 Play 按钮即可在编辑器中运行测试

### 3.4 构建Windows可执行文件

1. 在 Unity 菜单栏选择 `File → Build Settings`
2. 将以下场景添加到构建列表（按顺序）：
   - `Assets/Scenes/Login.unity`
   - `Assets/Scenes/MainMenu.unity`
   - `Assets/Scenes/Game.unity`
   - `Assets/Scenes/Leaderboard.unity`
3. 选择平台为 `Windows` (`PC, Mac & Linux Standalone` → `Windows`)
4. 设置架构为 `x86_64`
5. 点击 `Build`，选择输出目录（如 `e:\Miracle\GoldFisher\Build`）
6. 等待构建完成

### 3.5 运行客户端

构建完成后，在输出目录运行 `GoldFisher.exe`。

## 4. 配置说明

### 4.1 服务器IP端口配置

#### 服务器端
修改 `FishServer/Config/serverconfig.json`：
```json
{
  "host": "0.0.0.0",
  "port": 8888
}
```

#### 客户端
修改 `FishApp/Assets/Config/ServerConfig.json`：
```json
{
  "host": "192.168.1.100",
  "port": 8888
}
```

> 客户端的 `host` 必须填写服务器的实际IP地址。本地测试用 `127.0.0.1`，局域网测试用服务器内网IP，公网部署用服务器公网IP。

### 4.2 防火墙配置

服务器所在机器需开放对应TCP端口：

**Windows:**
```powershell
# 以管理员身份运行 PowerShell
New-NetFirewallRule -DisplayName "GoldFisher Server" -Direction Inbound -Protocol TCP -LocalPort 8888 -Action Allow
```

**Linux (ufw):**
```bash
sudo ufw allow 8888/tcp
```

### 4.3 数据目录

服务器运行时会在工作目录下创建 `data/` 目录存储玩家数据：

```
FishServer/
└── data/
    ├── players/
    │   ├── p_10001.json
    │   ├── p_10002.json
    │   └── ...
    └── leaderboard.json
```

> 请确保服务器进程对 `data/` 目录有读写权限。

## 5. 部署场景

### 5.1 本地开发
- 服务器和客户端都在本机运行
- 客户端配置 `host: 127.0.0.1`
- 适合开发和调试

### 5.2 局域网测试
- 服务器运行在一台机器上
- 客户端运行在其他机器上，配置 `host` 为服务器内网IP
- 确保服务器防火墙开放端口

### 5.3 公网部署
- 服务器部署在云服务器上
- 客户端配置 `host` 为服务器公网IP或域名
- 需在云服务器安全组中开放TCP 8888端口

## 6. 常见问题

### Q1: 客户端连接不上服务器？
- 检查服务器是否已启动
- 检查客户端 `ServerConfig.json` 中的IP和端口是否正确
- 检查服务器防火墙是否放行端口
- 检查服务器和客户端是否在同一网络（局域网测试时）

### Q2: 服务器启动报端口占用？
- 检查端口是否被其他程序占用：`netstat -ano | findstr 8888`
- 修改 `serverconfig.json` 中的 `port` 为其他可用端口
- 同时修改客户端配置

### Q3: Unity打开项目报错？
- 确认使用的是 2022.3.34f1c1 版本
- 尝试删除 `Library/` 目录后重新打开
- 检查 `Packages/manifest.json` 是否完整

### Q4: 服务器数据丢失？
- 检查 `data/` 目录是否存在且可写
- 检查 `serverconfig.json` 中 `dataDir` 配置
- 建议定期备份 `data/` 目录
