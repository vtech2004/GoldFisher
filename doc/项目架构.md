# 黄金钓鱼 - 项目架构文档

## 1. 整体架构

### 1.1 架构概览
本项目采用**客户端-服务器（C/S）**架构：

```
┌─────────────────────────────────────────────────────┐
│                    客户端 (FishApp)                   │
│                  Unity 2022.3.34f1c1                 │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐  │
│  │ Gameplay │  │    UI    │  │     Network      │  │
│  │  核心玩法 │  │   界面   │  │     网络通信     │  │
│  └──────────┘  └──────────┘  └────────┬─────────┘  │
└────────────────────────────────────────┼────────────┘
                                         │ TCP (JSON)
                                         │
┌────────────────────────────────────────┼────────────┐
│                 服务器 (FishServer)     │            │
│                C# .NET 8.0 控制台应用    │            │
│  ┌──────────────┐  ┌────────────────┐  │            │
│  │  GameServer  │  │ ClientSession  │←─┘            │
│  │   服务器核心  │←→│   会话管理     │               │
│  └──────┬───────┘  └────────────────┘               │
│         │                                            │
│  ┌──────┴───────┐  ┌────────────────┐               │
│  │   Managers   │  │    Storage     │               │
│  │  业务管理器  │→│    持久化层    │               │
│  └──────────────┘  └────────────────┘               │
└─────────────────────────────────────────────────────┘
```

### 1.2 技术栈
| 组件 | 技术 | 说明 |
|------|------|------|
| 客户端 | Unity 2022.3.34f1c1 | C# 开发 |
| 服务器 | .NET 8.0 控制台应用 | C# 开发 |
| 通信协议 | TCP + JSON | 换行符分隔消息 |
| 持久化 | JSON 文件 | 轻量级存储 |

### 1.3 通信方式
- 客户端与服务器通过 TCP 长连接通信
- 消息格式为 JSON，采用换行符 `\n` 分隔每条消息
- 详见 [网络协议.md](./网络协议.md)

### 3.2 服务器模块说明

#### GameServer - 服务器核心

`GameServer.cs` 是服务器的主入口类，职责：

- 初始化并启动 TCP 监听
- 管理服务器生命周期（启动/停止）
- 协调各子模块初始化
- 接受新连接并创建 `ClientSession`
- 主循环处理网络事件

#### ClientSession - 会话管理

`ClientSession.cs` 代表一个客户端连接，职责：

- 管理单个TCP连接的读写
- 按长度前缀分帧解析消息
- 将收到的消息分发到 `MessageDispatcher`
- 维护会话状态（是否已登录、关联的玩家ID）
- 连接断开时清理资源

`SessionManager.cs` 管理所有活跃会话：
- 新增/移除会话
- 按玩家ID查找会话
- 广播消息（如需要）

#### Managers - 业务管理

| 管理器 | 职责 |
|--------|------|
| `PlayerManager.cs` | 玩家管理：创建/查询玩家、维护在线玩家列表、玩家数据缓存 |
| `LevelManager.cs` | 关卡管理：关卡配置校验、解锁判断、关卡结果验证 |
| `ScoreManager.cs` | 计分管理：计算关卡总分、时间奖励、累计积分更新 |
| `LeaderboardManager.cs` | 排行榜管理：维护全服积分排名、快速查询TopN和自身排名 |

#### Storage - 持久化

| 类 | 职责 |
|----|------|
| `IStorage.cs` | 存储接口，定义数据读写契约 |
| `JsonStorage.cs` | JSON文件存储实现，将数据序列化为JSON文件持久化 |
| `PlayerData.cs` | 玩家数据模型（ID、名称、总积分、关卡记录等） |
| `LevelRecord.cs` | 关卡记录模型（关卡号、最高分、通关状态、游玩次数） |

存储文件示例（`data/players/p_10001.json`）：
```json
{
  "playerId": "p_10001",
  "playerName": "FisherKing",
  "totalScore": 4200,
  "highestLevel": 2,
  "levelRecords": [
    { "level": 1, "bestScore": 700, "passed": true, "playCount": 3 }
  ],
  "lastLogin": "2026-06-25 14:30:00"
}
```

## 4. 通信流程图

### 4.1 登录流程
```
客户端                          服务器
  │                               │
  │── TCP Connect ────────────────→│ TcpServer.Accept()
  │                               │ 创建 ClientSession
  │                               │
  │── login {playerName} ────────→│ ClientSession.OnReceive()
  │                               │   → MessageDispatcher.Dispatch()
  │                               │   → LoginMessage.Handle()
  │                               │   → PlayerManager.LoginOrRegister()
  │                               │   → Storage.Save()
  │                               │
  │←── login_resp {playerId,...} ─│ ClientSession.Send()
  │                               │
```

### 4.2 关卡结果上报流程
```
客户端                          服务器
  │                               │
  │── level_result {level,score} →│ MessageDispatcher
  │                               │   → LevelResultMessage.Handle()
  │                               │   → LevelManager.ValidateLevel()
  │                               │   → ScoreManager.UpdateScore()
  │                               │   → LeaderboardManager.UpdateRank()
  │                               │   → Storage.Save()
  │                               │
  │←── level_result_resp ────────│ {totalScore, rank}
  │                               │
```

### 4.3 排行榜查询流程
```
客户端                          服务器
  │                               │
  │── leaderboard {topN:10} ────→│ MessageDispatcher
  │                               │   → LeaderboardMessage.Handle()
  │                               │   → LeaderboardManager.GetTopN()
  │                               │   → LeaderboardManager.GetRank()
  │                               │
  │←── leaderboard_resp ─────────│ {entries, myRank}
  │                               │
```

## 5. 关键设计说明

### 5.1 客户端单例管理
- `GameManager`、`NetworkManager`、`UIManager` 采用 Unity 单例模式
- 在场景初始化时创建，跨场景不销毁（DontDestroyOnLoad）

### 5.2 服务器线程模型
- 服务器采用异步IO（async/await + TcpClient）
- 每个 `ClientSession` 独立处理读写，互不阻塞
- 共享数据（如排行榜）通过加锁或 ConcurrentCollection 保护

### 5.3 消息处理流程
```
TCP接收字节流
    ↓
长度前缀分帧
    ↓
JSON反序列化为 MessageBase
    ↓
MessageDispatcher 根据 type 路由
    ↓
对应 MessageHandler 处理业务逻辑
    ↓
Storage 持久化（如需要）
    ↓
构造响应消息
    ↓
JSON序列化 + 长度前缀
    ↓
TCP发送
```

### 5.4 数据一致性
- 玩家数据在关键操作（登录、关卡上报）后立即持久化
- 排行榜在内存中维护有序结构，定期同步到磁盘
- 服务器重启时从存储层重建内存数据
